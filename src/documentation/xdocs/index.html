Dido - Data In Data Out

Overview

Dido provides a utiity to map Java to and from external data formats such as CSV and fixed with flat files, or Excel.

Why yet another such framework when there are already so many? Here are Dido's requirements.

  • The data specification must be two way. A dido:plan created by the developer provider by the developer can be given to the developers reading data downstream so they can use it immediately in their applications. No more file specifications that drift out of date or are ambiguous. The plan is the plan.
  • The plan must not contain Java bean mapping information. Again, this is to facilitate the exchange of plans. One developers Apple type is another developers Fruit type. Most other mapping frameworks mix Java mapping information and data layout information.

Installing and Using Dido

Dido is an extension to the Oddjob task execution framework. To install Dido unzip the dido-n.n.n.zip file over your existing oddjob directory. After instalation the Oddjob directories should have been modified to look something like this. 


oddjob
  docs
    dido
      index.html
        api
          ...
        reference
          index.html
          ...
        dido-reference
          index.html
          ...
  opt
  	lib
  	   dido-n.n.n.jar
  oddballs
    oj-poi
       lib
         oj-poi-n.n.n.jar
         poi-x.x-yyyyyMMdd.jar   
         other poi jars...

The oj-poi Oddball is an optional extension to Dido for reading and writing Microsoft Excel files. The oj-poi directory and it's contents may be deleted if reading and writing to Excel is not required.

When you next start Oddjob, the Dido job definitions will be available to use in your Oddjob configurations.
